Overview

Industrial manipulation requires a heterogeneous skill set. Operators routinely turn valves, throw breakers, thread fasteners, drive screws, press buttons, open and close enclosures, and execute multi-step assembly procedures. Each of these actions involves a different combination of perception, contact reasoning, and force control.

HiveBoard is a low-barrier physical benchmark that captures this diversity in a way any laboratory can reproduce on its own hardware. A hexagonal honeycomb base accepts 13 interchangeable attachments grouped into three categories. Each attachment is printed as a single piece with a hexagonal footprint that mates with the base by press-fit, no fasteners required.

Every attachment ships with simulation-ready CAD, so the same task is available in real and simulated environments. The full benchmark, the protocol, and the trial logging template are released under a permissive license.

Get Started

Three steps to a working HiveBoard in your lab.

1

Download the STL files

Grab the latest CAD release from the project repository. The full library prints in a few hundred grams of PLA filament on a consumer-grade FDM printer.

Download STL →
2

Print with the recommended profile

PLA, 0.4 mm nozzle, 0.20 mm layer height, 4 perimeter walls, per-part-type infill (20–30%), 50 mm/s. Detailed orientation tips and per-part settings are in the README.

Print profile →
3

Run the protocol

Five recorded trials per attachment per platform. Log results in trials.xlsx (frozen header, dropdowns, color-coded categories) or the plain trials.csv. Submit your archive for cross-laboratory comparison.

Read the protocol →

Evaluation Protocol

The protocol evaluates a manipulation system, defined as an end-effector (gripper or hand) plus a control interface. The interface may be a teleoperated arm with a standard input device, or a wearable interface or exoskeleton driving the end-effector directly.

For each attachment, the operator runs unrecorded familiarization trials until completion times stabilize, then records 5 trials. Each trial is logged with outcome, completion time, attempts, regrasps, and (for the composed assembly attachments) the highest stage reached within the timeout.

The full per-attachment success criteria, timeouts, and stage definitions are documented in PROTOCOL.md.

What you get

  • PROTOCOL.md — full procedure
  • HOW_TO_FILL_TRIALS.md — column-by-column instructions
  • trials.xlsx — styled Excel template
  • trials.csv — plain CSV for scripts

Simulation Assets

Every attachment ships with articulated CAD that imports directly into common simulators.

URDF

For ROS-based simulators, MuJoCo, and PyBullet

USD

For Isaac Sim and Omniverse workflows

Articulated joints

Revolute (valves, breaker, lock), helical (threads), prismatic (drawer), free-body with collision proxies (shock-absorber pin)

Mass & inertia

Estimated from printed-part volume and PLA density, ready for physics simulation

Simulation assets are uploaded as students complete the URDF and USD exports. Watch the repository for releases.
